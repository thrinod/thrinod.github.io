Getting Started with GitHub Pages

📅 April 28, 2026 ⏱️ 8 min read 👤 Thrinod
GitHub Web Development Tutorial

Introduction

GitHub Pages is a free web hosting service provided by GitHub that allows you to host static websites directly from your GitHub repository. Whether you want to create a personal portfolio, blog, documentation, or project showcase, GitHub Pages is an excellent choice.

In this guide, I'll walk you through the process of setting up your first GitHub Page and deploying a website in just a few minutes.

Prerequisites

Before getting started, you'll need:

  • A GitHub account (free)
  • Basic knowledge of HTML and CSS (optional but helpful)
  • Git installed on your computer (optional if using GitHub web interface)

Step 1: Create a New Repository

The first step is to create a new repository on GitHub. For a user or organization site, the repository name is crucial:

  • For user sites: username.github.io
  • For organization sites: organizationname.github.io
  • For project sites: Any repository name will work
⚠️ Important: Make sure the repository name matches your GitHub username exactly for user sites to work properly.

Step 2: Enable GitHub Pages

Once your repository is created, you need to enable GitHub Pages:

  1. Go to your repository settings
  2. Scroll down to the "GitHub Pages" section
  3. Under "Source", select the branch you want to use (usually main or master)
  4. Select the folder (usually root /)
  5. Click Save

Step 3: Create Your First Page

GitHub Pages will look for an index.html file in your repository's root directory. Here's a simple starter template: 

<!DOCTYPE html>
<html>
<head>
    <title>My Website</title>
</head>
<body>
    <h1>Hello World!</h1>
    <p>Welcome to my GitHub Pages site.</p>
</body>
</html>

Step 4: Commit and Push

If you're using Git from your command line: 

git add index.html
git commit -m "Initial commit: Add index.html"
git push origin main

Step 5: Visit Your Site

After a few moments (usually 1-2 minutes), your site will be live at: 

https://username.github.io

(Replace "username" with your actual GitHub username)

Using Jekyll with GitHub Pages

GitHub Pages has built-in support for Jekyll, a popular static site generator. You can use Jekyll themes to quickly style your site without writing CSS.

To use a Jekyll theme, add a _config.yml file to your repository: 

theme: jekyll-theme-minimal

Custom Domain

You can connect your own custom domain to your GitHub Pages site. To do this, add a CNAME file to your repository with your domain name, and update your domain's DNS settings to point to GitHub Pages.

Conclusion

GitHub Pages is a powerful and free solution for hosting static websites. In just a few steps, you can have your personal site, blog, or portfolio live on the internet. The best part? It integrates seamlessly with your GitHub workflow!

Ready to get started? Create your repository now and build your online presence!
